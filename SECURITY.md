# Security policy

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could expose credentials, personal data, private evidence, or a deployed system. Report it privately to `hi@hadox.org` with the affected component, impact, reproduction steps, and any suggested mitigation.

Do not include live credentials or unnecessary personal data in the report. Hadox will acknowledge credible reports and coordinate a responsible disclosure timeline.

## Supported scope

This repository is an open research reference. Security fixes for the current default branch are accepted. Deployment-specific configuration and excluded private systems are outside this repository's support boundary.
